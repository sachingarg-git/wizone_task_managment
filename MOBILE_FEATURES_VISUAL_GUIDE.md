### How Sync Works

1. **User logs in on APK**
   - Session created on server
   - Token stored in device

2. **Fetches task list**
   - GET /api/tasks/my-tasks
   - Server returns tasks for logged-in user
   - Displayed in card view

3. **User updates task status**
   - PUT /api/tasks/{id}
   - Server saves to database
   - History entry created

4. **Web portal auto-refreshes**
   - Polls server every 30 seconds
   - Shows updated status immediately

5. **User uploads file in APK**
   - POST /api/tasks/{id}/files
   - File saved to server disk
   - Path stored in database
   - History entry with file link

6. **Changes visible everywhere**
   - APK: Manual refresh or auto-sync
   - Web: Auto-refresh every 30s
   - Both show same data
